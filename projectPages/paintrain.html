"Pain Train" the Video Game

What is this?

This is a video game I created for a class called Software Design. The goal was to use Python's Pygame library and an MVP (model-viewer-player) framework to create a fun experience for the player.

It started off as a mini partner project with Diego Garcia which I then adapted into a final project. Together, we created and implemented the concept of a platformer where the player will tragically always lose. I took it further and added (FOR EXAMPLE, THIS.falling, obstacles, the start screen, and better graphics) to make it a more polished game.

Why does the player always lose?

Sometimes, life is pain. Personally, I'm a fan of using unconventional video game mechanics to express that. None of this "escape-from-reality" stuff-- this is an opportunity to embrace those tough feelings.

How does it work?

There is a beautiful README that explains this on my github, which you can view here.

Fundamentally, it is a python file which contains lots of classes designed to fit in the category of either model, viewer, or controller. These serve different functions; model classes contain information about the object such as size, position, or shape, viewer classes draw the object on the screen, and the controller class modifies the previous types when certain events (such as key presses) occur.
